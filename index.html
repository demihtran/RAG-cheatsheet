01

Mental Model

RAG = retrieval system + generation policy, không chỉ là vector DB
1. LoadPDF, HTML, Markdown, DB, ticket, Slack, API.
2. CleanBỏ boilerplate, nav, duplicate, OCR noise.
3. ChunkChia tài liệu thành đơn vị vừa đủ ngữ cảnh.
4. EmbedBiến chunk/query thành vector cùng không gian.
5. RetrieveTìm top_k, filter metadata, rerank.
6. GenerateTrả lời dựa trên context, trích nguồn, từ chối khi thiếu bằng chứng.

Ý chính: model không "biết" tài liệu của bạn. RAG đưa đoạn liên quan vào context để model trả lời có căn cứ.

Lỗi phổ biến: chỉ dùng vector search rồi hy vọng LLM tự sửa mọi lỗi. Retrieval kém thì generation vẫn kém.

TL

When To Use

RAG, fine-tuning, long context
  • Dùng RAGDữ liệu thay đổi, cần citation, cần quyền truy cập theo user, cần giảm hallucination.
  • Dùng fine-tuneCần style/format/hành vi ổn định, không phải nhồi kiến thức private thường xuyên đổi.
  • Dùng long contextTập tài liệu nhỏ, ít query, cần reasoning xuyên toàn bộ file, latency/cost chấp nhận được.
  • Kết hợpRetrieve ít nhưng đúng, đưa vào context dài hơn, sau đó dùng eval để chọn cấu hình.

03

Chunking

Chọn ranh giới theo ngữ nghĩa trước, token sau
Chiến lượcDùng khiTradeoff
Fixed-sizeText đơn giản, cần baseline nhanh.Dễ cắt giữa ý; cần overlap.
RecursiveMarkdown, docs, prose có đoạn rõ.Baseline tốt nhất cho nhiều app.
SemanticQuery dài, tài liệu nhiều chủ đề.Tốn embed/split hơn, khó debug hơn.
Parent-childMuốn search đoạn nhỏ nhưng trả context lớn.Cần lưu quan hệ chunk cha/con.
Late chunkingModel embedding hỗ trợ context dài.Giữ ngữ cảnh tốt nhưng pipeline phức tạp.

Rule of thumb: bắt đầu với 300-800 token/chunk, overlap 50-150 token, rồi tune bằng retrieval eval.

Chunk tốt: tự đứng được, có heading/metadata, không quá dài làm loãng similarity.

04

Embedding & Index

Vector là recall layer, không phải truth layer
  • Embedding modelChọn theo ngôn ngữ, domain, dimension, cost, latency, max input.
  • DistanceCosine, dot product, L2; normalize vector nếu metric yêu cầu.
  • ANN indexHNSW/IVF giúp query nhanh hơn brute force khi dữ liệu lớn.
  • Metadata filterLọc trước hoặc trong retrieval theo tenant, ACL, date, product, language.
  • Re-embedKhi đổi model embedding, phải build index mới; không trộn vector khác model.
  • FreshnessThiết kế incremental indexing, delete/update tombstone, background jobs.

Retrieval Patterns

Tăng recall trước, tăng precision sau
PatternMục tiêuGhi chú

Keyword/BM25Tìm exact match.Tốt cho tên API, SKU, error code, điều khoản pháp lý.

06

Minimal RAG Pipeline

Pseudo-code framework-agnostic
# Offline indexing
documents = load_documents(["docs/", "wiki/", "tickets.csv"])
documents = clean_and_normalize(documents)
chunks = split_documents(
    documents,
    strategy="recursive",
    chunk_size=600,
    chunk_overlap=100,
    preserve_headings=True,
)
vectors = embed(chunks.text)
vector_store.upsert(ids=chunks.ids, vectors=vectors, metadata=chunks.metadata)

# Online retrieval + generation
def answer(question, user):
    rewritten = rewrite_query(question)

    candidates = hybrid_search(
        dense_query=embed(rewritten),
        keyword_query=rewritten,
        filters={"acl": user.groups},
        top_k=40,
    )

    ranked = rerank(query=question, documents=candidates, top_k=6)
    context = format_context(ranked, include_citations=True)

    return llm.generate(
        system=RAG_SYSTEM_PROMPT,
        user={"question": question, "context": context},
        temperature=0,
    )

Production: cache embedding/query, timeout retrieval, log trace, lưu citation chunk ids.

Security: filter ACL trước khi generate. Không bao giờ retrieve toàn bộ rồi mới che output.

Dùng khi baseline đã đo được

A1

HyDE Query Expansion

Generate hypothetical answer, embed it, then retrieve
Khái niệm HyDE tạo câu trả lời giả định rồi dùng text đó làm retrieval query giàu ngữ nghĩa hơn.
Khi dùng Query quá ngắn, nhiều paraphrase, corpus không dùng đúng wording của user.
Tradeoff Tăng recall nhưng thêm một LLM call và có thể drift intent.
Guardrail Retrieve bằng cả query gốc và hypothetical answer, rồi merge/dedupe.
def hyde_retrieve(question, user):
    hypo = llm.generate(
        system="Write a short factual passage that would answer the question.",
        user=question,
        temperature=0,
    )

    # Retrieve from original intent and hypothetical document.
    dense_a = vector_store.search(embed(question), top_k=20, filters=acl(user))
    dense_b = vector_store.search(embed(hypo), top_k=20, filters=acl(user))

    return dedupe_by_chunk_id(dense_a + dense_b)

Best: dùng HyDE như recall booster, sau đó rerank bằng query gốc.

Mistake: chỉ retrieve bằng hypothetical text, làm retrieval lệch khỏi intent thật.

A2

RAG-Fusion / RRF

Multi-query retrieval + Reciprocal Rank Fusion
Khái niệm Sinh nhiều query biến thể, retrieve riêng, merge ranking bằng RRF.
Khi dùng Câu hỏi rộng, user wording kém, cần recall cao hơn vector search đơn.
Parameter num_queries, top_k_each, rrf_k.
Cost Tăng số lượt search; nên cache query variants.
def rrf_merge(ranked_lists, k=60):
    scores = {}
    for docs in ranked_lists:
        for rank, doc in enumerate(docs, start=1):
            scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)
    return sorted(scores, key=scores.get, reverse=True)

queries = llm.generate_query_variants(question, n=4)
runs = [hybrid_search(q, top_k=20) for q in [question] + queries]
candidate_ids = rrf_merge(runs)[:40]

Best: fusion để tạo candidate pool, không thay thế reranker cuối.

Mistake: sinh query quá tự do làm thêm topic mới không có trong intent.

A3

Parent-Child Retrieval

Search small chunks, return larger parent context
Khái niệm Embed chunk nhỏ để match chính xác, nhưng trả parent section để model có đủ context.
Khi dùng Docs dài có heading/section rõ; câu trả lời cần đoạn trước/sau.
Index Child chunks có parent_id; parent store giữ full section.
Risk Parent quá lớn làm context noise; cần cap token.
def index_parent_child(doc):
    for section in split_by_heading(doc):
        parent_id = stable_hash(section.title + section.text)
        parent_store.put(parent_id, section.text, section.metadata)

        for child in split_small(section.text, size=250):
            vector_store.upsert(
                id=stable_hash(parent_id + child.text),
                vector=embed(child.text),
                metadata={**section.metadata, "parent_id": parent_id},
            )

def retrieve_context(question):
    children = vector_store.search(embed(question), top_k=12)
    return [parent_store.get(c.metadata["parent_id"]) for c in children]

Best: dedupe parent ids và chỉ giữ parent sections có child score cao.

Mistake: trả cả document parent rất lớn làm model đọc nhầm đoạn.

A4

Reranking & Context Compression

Retrieve broad, rerank precise, compress before prompt
Rerank Cross-encoder/LLM đánh giá query-document pair chính xác hơn vector distance.
Compression Trích câu/đoạn liên quan từ chunk dài để giảm token và noise.
Khi dùng Corpus lớn, top_k cao, docs nhiều phần không liên quan.
Budget Candidate pool 30-100, final context 4-10 chunks tùy app.
def retrieve_rerank_compress(question, user):
    candidates = hybrid_search(
        query=question,
        filters=acl(user),
        top_k=60,
    )

    ranked = cross_encoder.rerank(
        query=question,
        documents=candidates,
        top_k=8,
    )

    compressed = [
        extract_relevant_sentences(question, doc.text, max_tokens=180)
        for doc in ranked
    ]
    return format_context(compressed)

Best: rerank bằng query gốc, không chỉ query rewritten.

Mistake: compression xóa mất citation hoặc câu phủ định quan trọng.

A5

Self-Query Metadata Filters

LLM extracts structured filters from natural language
Khái niệm Tách câu hỏi thành semantic query + filters như date, product, region, acl.
Khi dùng Corpus nhiều metadata; user hỏi "policy năm 2025 của EU".
Guardrail Validate filters theo allowlist schema, không cho filter tự do.
Security ACL/tenant filter luôn do backend thêm, không để LLM quyết định.
FilterSchema = {
  "product": ["billing", "security", "support"],
  "year": "integer between 2020 and 2026",
  "region": ["US", "EU", "APAC"],
}

parsed = llm.structured_extract(
    schema=FilterSchema,
    text="EU security policy from 2025",
)

filters = validate_allowlist(parsed.filters)
filters.update(acl(current_user))

docs = vector_store.search(
    query=embed(parsed.semantic_query),
    filters=filters,
    top_k=20,
)

Best: index metadata đầy đủ từ ingestion; filter trước rerank.

Mistake: để LLM tạo raw SQL/filter expression không validate.

A6

Graph RAG

Use entities and relationships when similarity alone is not enough
Khái niệm Trích entity/relation từ docs, tạo graph, retrieve subgraph + text chunks.
Khi dùng Câu hỏi multi-hop: ai liên quan tới cái gì, nguyên nhân, chuỗi sự kiện.
Hybrid Graph retrieval để tìm facts liên kết; vector retrieval để lấy evidence text.
Risk Entity extraction sai tạo graph sai; cần provenance và confidence.
def graph_rag_answer(question):
    entities = entity_linker.extract(question)

    subgraph = graph_db.expand(
        seeds=entities,
        hops=2,
        edge_types=["owns", "depends_on", "mentions"],
        limit=80,
    )

    evidence_ids = [edge.source_chunk_id for edge in subgraph.edges]
    text_evidence = doc_store.get_many(evidence_ids)

    context = format_graph(subgraph) + format_context(text_evidence)
    return grounded_generate(question, context)

Best: luôn giữ source chunk cho mỗi edge/entity để câu trả lời cite được.

Mistake: cho graph triples không có bằng chứng text vào prompt như sự thật tuyệt đối.

A7

CRAG / Self-RAG Loop

Grade retrieval, retry, or abstain when context is weak
Khái niệm Model/evaluator chấm context có đủ trả lời không; nếu không thì rewrite/search lại.
Khi dùng Support/legal/medical style QA cần nói "không biết" thay vì bịa.
Signals Retrieval score, rerank score, coverage, contradiction, citation support.
Limit Cap số lần retry để tránh loop tốn chi phí.
def corrective_rag(question):
    for attempt in range(2):
        docs = retrieve(question)
        grade = judge_context(
            question=question,
            docs=docs,
            rubric=["relevant", "complete", "non_conflicting"],
        )
        if grade.is_answerable:
            return grounded_generate(question, docs)
        question = rewrite_query(question, missing=grade.missing_facts)

    return "I do not have enough reliable context to answer."

Best: log grade reason và missing facts để cải thiện ingestion/retrieval.

Mistake: dùng cùng LLM vừa generate vừa grade mà không có rubric rõ.

A8

Multi-Hop Agentic RAG

Plan, search, read, refine for complex questions
Khái niệm Agent lặp search/read khi câu hỏi cần nhiều bằng chứng hoặc nhiều nguồn.
Khi dùng Comparison, root-cause, "why", "how changed over time".
Tools search_docs, open_source, cite, ask_clarification.
Guardrail Max steps, max tokens, required citations, no-answer path.
def agentic_rag(question):
    notes = []
    for step in range(5):
        action = planner.next_action(question=question, notes=notes)

        if action.type == "search":
            notes.extend(retrieve(action.query))
        elif action.type == "read":
            notes.append(open_chunk(action.chunk_id))
        elif action.type == "answer":
            return grounded_generate(question, notes)

    return grounded_generate(question, notes, require_citations=True)

Best: dùng agentic RAG cho câu hỏi khó, không phải mọi câu hỏi đơn giản.

Mistake: để agent search vô hạn hoặc không trace từng action.

Evaluation

Đo retrieval và generation riêng, rồi đo end-to-end
MetricĐo cái gìCách dùng
Recall@kChunk đúng có nằm trong top k không.Tune chunking, hybrid weight, top_k.
MRR / nDCGChunk đúng đứng cao đến đâu.Đánh giá ranking/reranker.
Context precisionContext đưa vào có nhiều phần thật sự liên quan không.Giảm noise, giảm hallucination.
FaithfulnessCâu trả lời có được hỗ trợ bởi context không.Phát hiện model bịa hoặc suy quá xa.
Answer relevanceCâu trả lời có đúng intent câu hỏi không.Phát hiện trả lời lan man.
Citation accuracyNguồn dẫn có thật sự hỗ trợ claim không.Cần cho chatbot tài liệu, legal, support.

Dataset: tạo 50-200 câu hỏi vàng từ logs, docs quan trọng, edge cases và câu hỏi không có đáp án.

Regression: mỗi lần đổi chunk/model/top_k/prompt phải chạy lại eval và so sánh cost/latency.

10

Failure Modes

Debug theo từng layer
  • No answerChunk quá lớn/nhỏ, query mismatch, filter sai, index stale.
  • Wrong answerRetrieve nhầm version, context noise, prompt không bắt citation.
  • HallucinationContext thiếu, model dùng prior knowledge, temperature cao.
  • SlowTop_k quá lớn, rerank đắt, không cache, index chưa tối ưu.
  • Leak dataACL filter sau retrieval, metadata thiếu tenant/user group.
  • Prompt injectionRetrieved doc chứa lệnh độc hại làm model bỏ policy.

Production Checklist

Những thứ cần có trước khi demo thành sản phẩm
  • Data contractĐịnh nghĩa nguồn dữ liệu, owner, update frequency, retention.
  • Access controlTenant/user/group filter nằm trong retrieval path.
  • Index jobsIncremental update, retry, idempotency, delete handling.
  • ObservabilityTrace load -> split -> retrieve -> rerank -> generate; log token/cost/latency.
  • Eval suiteGolden questions, no-answer tests, regression threshold.
  • CitationsLưu source id, page/section, chunk id; UI cho người dùng mở nguồn.
  • FallbacksKhi context yếu: ask clarification, say unknown, escalate to search/tool.
  • Cost controlsCache embeddings, cap top_k, budget rerank, summarize context khi cần.
  • SafetyChặn prompt injection, PII leakage, unsafe tool calls, data exfiltration.
